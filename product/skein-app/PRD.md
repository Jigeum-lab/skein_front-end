# Skein — 제품 요구사항 문서 (PRD)

> v1.1 변경 요약(DA 비평 반영): 비주얼 생성 P0→P1 강등 / P0 코어를 "룸→카피→라이브러리 Approved" 검증 루프로 재정의 / KPI 계측 스키마를 P0 데이터 요건으로 추가 / 인증 완료 기준을 fail-closed 리다이렉트로 명시 / 핵심 가정 검증 장치 표 신설(§5-D) / RLS deny-by-default 원칙 명시 / 개방 질문에 blocking 구분 추가.

## 1. 제품 개요

**제품명**: Skein

**한 줄 설명**: 멀티브랜드 마케팅 대행사가 브랜드별 톤 암묵지를 명시적으로 관리하고, AI 카피 생성 시 브랜드 컨텍스트를 자동 주입받기 위한 콘텐츠 작업실.

**목표**:
- 비공개 베타에서 **핵심 가정 3종을 검증**(톤 학습 품질 / 통합 워크플로 수요 / 셋업 30분). 검증 장치는 §5-D 참조.
- 온브랜드 채택률(생성 안이 수정 없이 Approved되는 비율)의 **베이스라인 측정** — 측정용 데이터 스키마는 P0 요건(§4 라이브러리, §5-C).
- 시니어 AM의 톤 지식을 브랜드 룸으로 이전해 병목 제거 (Rachel Kim 시나리오 검증).

**비목표** (이번 베타 범위 제외):
- AI 비주얼 생성 — **P1로 이동** (P0 검증 루프엔 불필요. §3 참조).
- SNS 스케줄링·퍼블리싱 (콘텐츠 생성·승인까지만).
- 클라이언트 포털 (외부 클라이언트 직접 접근 UI).
- 자동화 워크플로 트리거 (Zapier 등 연동).
- 유료 결제·구독 관리 (베타 기간 해당 없음).
- 브랜드 성과 분석 **대시보드 UI** (P2). 단, 채택률 산출용 **이벤트/스키마는 P0** — 대시보드 없이 쿼리로 베이스라인 산출.
- 온브랜드 자동 검수(생성물이 가이드 부합 여부를 ML로 자동 판단) — 별도 ML 기능, P2.

## 4. 기능 상세 (P0)

### 인증 (Supabase Auth) — P0
완료 기준: 이메일+구글 OAuth 로그인, **미들웨어 fail-closed 차단**(`getUser()` null → `/login`, Supabase 키 부재 시에도 통과 금지), 세션 만료 처리.

**정상 흐름**:
1. 랜딩에서 "시작하기" → `/login`
2. 이메일+비밀번호 또는 구글 OAuth
3. 성공 → `@supabase/ssr` 세션 쿠키 → `/dashboard`
4. 세션 유효 시 재방문 시 로그인 생략
5. 로그아웃 → 쿠키 삭제 → `/login`

**엣지 케이스**:
- 미인증 `/dashboard` 직접 접근 → 미들웨어가 `/login` 리다이렉트
- 구글 OAuth 취소(팝업 닫기) → 에러 없이 로그인 화면 유지
- 이미 가입된 이메일로 구글 OAuth → [결정 필요: 계정 병합 정책]
- 비밀번호 분실 → Supabase 기본 재설정 링크
- 세션 만료 중 API 호출 → 401 후 자동 `/login`
- **Supabase 환경변수 누락 배포** → 전 라우트 차단(fail-closed), 에러 페이지 노출 (현 `middleware.ts`의 fail-open 동작을 반드시 교체)

**에러 처리**:
- Auth API 장애: 로그인 버튼 비활성화 + "인증 서비스에 일시적인 문제가 발생했습니다"
- 네트워크 단절: 폼 제출 실패 + 재시도 안내
- 타임아웃(3초 초과): 스피너 해제 + 재시도 버튼

**비기능**: 로그인 응답 [확인 필요: Supabase SLA]. 소셜 제공자 구글 (애플·깃허브는 [결정 필요]).

> **현 코드 갭(반드시 구현)**: `src/lib/supabase/middleware.ts`는 현재 `getUser()`로 토큰 갱신만 하고 리다이렉트가 없으며, 키 부재 시 `NextResponse.next()`로 전 라우트를 통과시킨다(fail-open). 이 완료 기준은 미구현 상태이며 인증은 KPI·격리보다 선행하는 보안 게이트다.

**비기능**: `brands` 테이블 `workspace_id` RLS 필수. 브랜드 상한 [결정 필요]. 목록 로드 [확인 필요].

### AI 카피 생성 — P0
완료 기준: 브랜드 룸 컨텍스트 **서버사이드 자동 주입**, 채널·포맷 선택, Draft 저장, 컨텍스트 부재 경고. **AI 제공자(#9) 확정이 착수 선결 조건.**

**정상 흐름**:
1. `/b/[brandId]/copy`
2. 채널(인스타·이메일·광고 등)·포맷(캡션·헤드라인·바디) 선택
3. 작업 지시 입력
4. "생성" → 룸 톤 가이드+레퍼런스를 **서버 API 라우트에서 조합**하여 AI 호출

### 콘텐츠 라이브러리 + KPI 계측 — P0
