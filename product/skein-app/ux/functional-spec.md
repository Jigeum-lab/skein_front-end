# Skein — 기능명세서 (Functional Spec)
버전: v1.0 | 날짜: 2026-06-14 | 기준: PRD v1.2, ia-userflow v1.1

> 형식: 기능 ID(F001~) · 입력 → 처리 → 출력 · 브레이크포인트(Mobile/PC) · 엣지케이스 4종.
> P0 코어 루프(룸→카피→라이브러리) + v1.2 신규(레퍼런스 자동추출·사용량 계측·가격) 중심. P1은 자리표시.

---

## F001 · 인증 (로그인/회원가입) — P0
| 항목 | 내용 |
|---|---|
| 입력 | 이메일·비밀번호 / 구글 OAuth |
| 처리 | Supabase Auth 세션 쿠키 발급(`@supabase/ssr`), 미들웨어 fail-closed 검증 |
| 출력 | 성공→`/dashboard` 리다이렉트, 실패→인라인 에러 |
| Mobile | 단일 컬럼 폼, 풀폭 버튼 |
| PC | 중앙 카드(max-w), 좌측 브랜드 비주얼 |
| 엣지 | ① 키 부재→전 라우트 차단 ② OAuth 취소→화면 유지 ③ 세션 만료→401→`/login` ④ Auth 장애→버튼 비활성+안내 |

## F002 · 멀티브랜드 워크스페이스 — P0
| 항목 | 내용 |
|---|---|
| 입력 | 워크스페이스명 / 브랜드명·색상·로고 |
| 처리 | 브랜드 CRUD, RLS `workspace_id` 격리, 사이드바 스위처 반영 |
| 출력 | 사이드바 브랜드 목록, `/b/{brandId}` 스코프 전환 |
| Mobile | 스위처 드롭다운, 사이드바 시트(off-canvas) |
| PC | 좌측 고정 사이드바 + 스위처 |
| 엣지 | ① 브랜드 0개→빈 상태 CTA ② 타 워크스페이스 ID 직접접근→404(RLS) ③ 삭제 확인 다이얼로그 ④ 브랜드명 중복(정책 #3) |

## F003 · 브랜드 룸 — 레퍼런스 수집 — P0 (v1.2 갱신)
| 항목 | 내용 |
|---|---|
| 입력 | ① 계정 연동(OAuth, P1) ② 링크 URL ③ 직접 텍스트 |
| 처리 | URL→`detectPlatform()`→어댑터 텍스트 추출(비동기 `pending→extracted`), `brand_references` 저장 |
| 출력 | 레퍼런스 카드 목록(플랫폼 배지·채널·상태·미리보기) |
| Mobile | 추가 폼 세로 스택, 카드 1열 |
| PC | 추가 폼 + 목록 2분할 가능, 카드 1열(max-w-4xl) |
| 엣지 | ① 비공개·미지원·자막없음→직접입력 폴백 ② 추출 지연→`pending` 스피너 ③ 중복 URL(정책) ④ 빈 입력→버튼 비활성 |

## F004 · 브랜드 룸 — 톤 프로필 추출 — P0 (v1.2 신규)
| 항목 | 내용 |
|---|---|
| 입력 | 레퍼런스 풀(≥3·채널 커버) |
| 처리 | LLM structured output→보이스 축·선호/금지 어휘·신뢰도, 사용자 검토·수정(HITL) |
| 출력 | 톤 프로필 카드 + "컨텍스트 활성화" 배지, `tone_profile` 확정 |
| Mobile | 보이스 축 바 세로 나열, 어휘 칩 줄바꿈 |
| PC | 동일(max-w-4xl), 카드 그룹 |
| 엣지 | ① 레퍼런스 부족→비활성+안내 ② 추출 저신뢰→재추출 ③ 사용자 수정→재확정 ④ 추출 API 실패→재시도 |

## F005 · AI 카피 생성 (`/write`) — P0
| 항목 | 내용 |
|---|---|
| 입력 | 채널·포맷 선택 + 작업 지시 |
| 처리 | 서버 라우트에서 `voice_summary`+top-K 레퍼런스 주입→AI 호출(#9 제공자), `usage_events`(copy=1크레딧) 기록 |
| 출력 | 결과 카드 → Draft 저장(`generated_text` 원본 보존) |
| Mobile | 폼 위·결과 아래 세로 |
| PC | 폼 좌·결과 우 2분할 가능 |
| 엣지 | ① 컨텍스트 부재→경고 배너(계속 가능) ② 지시 미입력→인라인 에러 ③ AI 장애/타임아웃→재시도 ④ 빈 결과→다시 시도 |

## F006 · 콘텐츠 라이브러리 + KPI — P0
| 항목 | 내용 |
|---|---|
| 입력 | 카드 상태 전환(리뷰요청·승인·재작업) |
| 처리 | Draft→Review→Approved, `edited_before_approval`·`revision_count`·`approved_at` 기록 |
| 출력 | 카드 목록·상세 모달(원본 vs 현재), Approved 텍스트 복사 |
| Mobile | 카드 1열, 모달 풀스크린 시트 |
| PC | 카드 그리드, 중앙 다이얼로그 |
| 엣지 | ① 콘텐츠 0개→빈 상태 ② 전환 실패→롤백+토스트 ③ Approved 후 재수정(정책 #16) ④ 100건+ 로딩(정책 #17) |

## F007 · 사용량 계측·표시 — P0 (v1.2 신규)
| 항목 | 내용 |
|---|---|
| 입력 | 과금 액션 발생(카피·톤추출·비주얼) |
| 처리 | `usage_events` 기록, 주기별 크레딧 집계(`CREDIT_COST`) |
| 출력 | 대시보드 요약 바(`UsageMeter compact`) + 설정의 유형별·브랜드별 분해 |
| Mobile | 바 + 통계 2열 그리드 |
| PC | 바 + 통계 3열 + 브랜드별 막대 |
| 엣지 | ① 사용 0→0% 표시 ② 80%↑→임박 경고(destructive) ③ 100%↑→소진(베타=경고만) ④ 주기 경계 리셋 |

## F008 · 플랜·청구 — P1 (v1.2 신규)
| 항목 | 내용 |
|---|---|

| PC | 플랜 카드 3열 |
| 엣지 | ① 현재 플랜→버튼 비활성 ② 베타→무과금 안내 ③ 다운그레이드 시 한도 초과 처리 ④ 결제 실패 |

## F009 · 멀티브랜드 전환 — P0
