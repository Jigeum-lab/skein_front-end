### 페르소나 1: Rachel Kim — 계정 매니저 (Account Manager)

**핵심 특성**: 동시에 4~6개 뷰티 브랜드를 담당. 클라이언트 기대치와 내부 팀 역량 사이의 중재자. 브랜드 톤의 수호자이지만 그 지식은 문서가 아닌 머릿속에 있다.

| 항목 | 내용 |
|---|---|
| 나이 | 29세 |
| 직책 | Senior Account Manager, 뷰티 D2C 전문 대행사 |
| 경력 | 5년 (뷰티 업계 3년) |
| 사용 기기 | MacBook Pro (주), iPhone (이동 중 승인), iPad (클라이언트 미팅) |
| 주요 채널 | Slack, 이메일, Notion, Google Docs, Figma (보기 전용) |

**Goals**
- 담당 브랜드마다 다른 톤/보이스를 팀 전체가 일관되게 유지하도록 관리
- 클라이언트 리비전 라운드를 2회 이내로 줄여 마진 보호
- 신입 팀원이 온보딩 후 2주 안에 온브랜드 콘텐츠를 낼 수 있도록 지원

**Frustrations**
- 브랜드 가이드라인이 PDF로 존재하지만 "살아있는 문서"가 아님 — 업데이트가 반영되지 않음
- 카피라이터가 브랜드 A의 톤으로 브랜드 B 카피를 쓰는 "톤 혼선" 빈발 [확인 필요: 실제 빈도]
- 클라이언트에게 "이건 우리 톤이 아니에요"라는 피드백을 받으면 왜 틀렸는지 팀에 설명하기 어려움 — 근거가 없음

**Behaviors**
- 새 브랜드 온보딩 시 직접 카피 예시를 Notion에 수작업으로 정리
- Slack DM으로 "이 브랜드는 이런 느낌으로 써줘"를 반복 전달
- 최종 카피 승인 전 본인이 직접 톤 검토 — 병목 발생

> [확인 필요] "저는 결국 브랜드마다 Notion 페이지를 하나씩 만들어서 '이건 되고 이건 안 돼' 예시를 넣어둬요. 그게 제 머릿속 지식을 유일하게 전달하는 방법이에요."
> (유사 패턴 출처: Shared Drive, 2026 — 전략이 문서 대신 사람 머릿속에만 존재하는 문제)

### 페르소나 2: Jordan Lee — 카피라이터 / 콘텐츠 마케터

**핵심 특성**: 주당 3~5개 브랜드의 카피를 생산. AI 툴을 이미 쓰고 있지만 브랜드별 톤 맥락 주입이 번거로워 결과물 품질이 들쭉날쭉. "빠르게 써야 하는 압박"과 "온브랜드여야 하는 요구" 사이에서 끊임없이 긴장.

| 항목 | 내용 |
|---|---|
| 나이 | 26세 |
| 직책 | Copywriter / Content Marketer |
| 경력 | 3년 (뷰티 업계 1.5년) |
| 사용 기기 | MacBook Air (주), iPhone |
| 주요 채널 | ChatGPT/Claude (카피 초안), Google Docs, Notion (브랜드 레퍼런스), Slack |

**Goals**
- 브랜드 톤을 매번 찾아보지 않고 빠르게 "온브랜드" 카피 초안 생성
- 디자이너에게 넘길 때 비주얼 방향을 명확하게 전달해서 수정 횟수 줄이기
- 결과물이 클라이언트에게 "이건 우리 목소리가 아니에요" 피드백 없이 통과되기

**Frustrations**
- AI로 카피를 뽑을 때마다 브랜드 가이드라인을 복사해서 프롬프트에 넣어야 함 — 반복 작업
- "Friendly but professional"같은 모호한 톤 가이드로는 실제로 무엇을 써야 할지 모름
- 브랜드 A 카피 작업 후 브랜드 B로 전환할 때 "톤 리셋"이 안 되어 실수 발생
- 완성된 카피를 디자이너에게 전달하면 레이아웃에서 텍스트가 잘리거나 디자인과 맞지 않음

**Behaviors**
- 작업 시작 전 Notion에서 해당 브랜드 페이지를 열어 예시 카피를 훑어봄 (약 10~15분 소요)
- 동일 브랜드 이전 카피를 Google Docs에서 검색해서 톤 참고
- 초안 완성 후 계정 매니저에게 Slack으로 "이 방향 맞죠?" 확인 요청 — 매니저 병목 유발

> [확인 필요] "브랜드 가이드라인이 있어도 실제로 어떻게 써야 하는지는 예시를 봐야 알아요. 근데 예시가 여기저기 흩어져 있어서 매번 찾는 게 더 오래 걸려요."
> (유사 패턴 출처: Amelie Pollak, 2024 — 모호한 브랜드 보이스 브리핑이 endless revision 유발)

**출처**: The Beauty Copywriters 서비스 페이지, Productive Shop 협업 분석, Amelie Pollak 브랜드 보이스 브리핑 가이드

### 페르소나 3: Maya Torres — 그래픽 디자이너

**핵심 특성**: 카피 없이 디자인을 시작해야 하는 상황이 반복됨. 브랜드별 비주얼 정체성을 유지하지만 카피↔비주얼 핸드오프가 비공식적이어서 최종 단계에서 레이아웃 붕괴가 잦음. 브랜드 룸 개념에 가장 즉각적 가치를 느낄 사용자.

| 항목 | 내용 |
|---|---|
| 나이 | 28세 |
| 직책 | Graphic Designer / Visual Content Creator |
| 경력 | 4년 (뷰티/CPG 업계 2년) |
| 사용 기기 | MacBook Pro + 외부 모니터, iPad Pro (스케치) |
| 주요 채널 | Figma (주), Adobe Creative Suite, Slack, Notion (레퍼런스 보드) |

**Goals**
- 카피 확정 전에도 레이아웃 작업을 시작할 수 있는 충분한 콘텐츠 맥락 확보
- 브랜드별 비주얼 에셋(색상, 폰트, 이미지 톤)을 한 곳에서 바로 참조
- 완성된 비주얼이 카피와 함께 클라이언트에게 "온브랜드"로 승인받기

**Frustrations**
- 카피라이터에게 최종 텍스트를 받으면 예상보다 길거나 짧아서 레이아웃 전면 수정
- 브랜드 A와 B의 비주얼 에셋이 섞이는 실수 — 특히 여러 브랜드를 동시 작업할 때
- 비주얼 방향을 결정할 때 카피의 감정적 톤을 참고해야 하는데 카피가 늦게 나옴
- 클라이언트가 "이 이미지는 우리 브랜드 느낌이 아니에요"라고 하는데 기준이 불명확

**Behaviors**
- Figma에 브랜드별 컴포넌트 라이브러리를 직접 구축해서 관리 (비공식, 개인 작업)
- 카피 없이 더미 텍스트로 먼저 레이아웃 잡고 나중에 실제 카피로 교체
- Notion에 브랜드별 무드보드 페이지를 수작업으로 유지

> [확인 필요] "카피가 얼마나 길지 모르면 레이아웃을 확정할 수가 없어요. 근데 카피가 나올 때까지 기다리면 일정이 밀려요. 그래서 결국 뭔가 맞지 않는 걸 나중에 고치는 게 루틴이 돼버렸어요."
> (유사 패턴 출처: Productive Shop, 2024 — copy-design handoff 구조 부재로 인한 후기 수정 비용)

**출처**: Productive Shop 카피-디자인 협업 분석, Smartsheet 크리에이티브 워크플로 가이드, BeautyMatter 2025 에이전시 분석

## 2. JTBD (Jobs To Be Done)

형식: When [상황/트리거], I want to [동기/목표], so I can [결과/가치].

### 기능적 잡 (Functional Jobs)

**F-1. 브랜드 톤 즉시 접근**
When 새 콘텐츠 작업을 시작할 때,
I want to 해당 브랜드의 톤·보이스 기준과 예시를 한 곳에서 바로 찾고 싶다,
so I can 다른 툴을 오가거나 동료에게 물어보는 시간 없이 즉시 온브랜드 작업을 시작할 수 있다.

**F-2. 온브랜드 카피 생성**
When AI로 카피 초안을 만들어야 할 때,
I want to 브랜드 가이드라인을 프롬프트에 매번 복사붙여넣기하지 않고 브랜드 컨텍스트가 자동으로 적용되길 원한다,
so I can 반복 작업 없이 빠르게 온브랜드 초안을 얻고 리비전 횟수를 줄일 수 있다.

**F-3. 카피-비주얼 동시 핸드오프**
When 카피가 확정되어 디자이너에게 넘길 때,
I want to 텍스트 길이·감정 톤·비주얼 방향을 함께 전달하고 싶다,
so I can 디자이너가 별도 브리핑 없이 레이아웃 작업을 바로 시작할 수 있다.

### 감정적 잡 (Emotional Jobs)

**E-1. "내가 틀리지 않았다"는 확신**
When 카피 초안을 제출하기 전에,
I want to 이 결과물이 해당 브랜드 톤에 맞는지 스스로 확인할 수 있길 원한다,
so I can 매니저나 클라이언트의 "이건 우리 톤이 아니에요" 피드백 없이 자신감 있게 제출할 수 있다.

**E-2. 멀티브랜드 전환 시 심리적 클린슬레이트**
When 하루에 여러 브랜드를 오가며 작업할 때,
I want to 현재 작업 중인 브랜드의 맥락에 완전히 집중할 수 있길 원한다,
so I can 다른 브랜드의 톤이 의도치 않게 섞이는 실수를 하지 않고 각 브랜드에 집중할 수 있다.

### 사회적 잡 (Social Jobs)

**S-1. 신입에게 브랜드 톤 전수**
When 신입 카피라이터나 프리랜서를 프로젝트에 투입해야 할 때,
I want to 내 머릿속에 있는 브랜드 지식을 구조화된 형태로 공유하고 싶다,
so I can "이 브랜드는 이런 느낌"이라는 암묵지를 팀 전체의 자산으로 만들고 신입 온보딩 시간을 단축할 수 있다.

**S-2. 클라이언트 신뢰 구축**
When 클라이언트에게 콘텐츠를 제출할 때,
I want to 매 라운드 결과물이 브랜드 가이드라인과 명확하게 연결된 근거와 함께 전달되길 원한다,
so I can "왜 이 방향인가"를 설명할 수 있어 클라이언트의 신뢰를 얻고 리비전 라운드를 줄일 수 있다.

### 특징적 행동 패턴 요약

1. **비공식 지식 저장소 의존**: 공식 시스템 없이 개인이 Notion 페이지·Figma 라이브러리를 수작업으로 만들어 브랜드 지식 유지. 해당 인물이 떠나면 지식 소실.
2. **매니저 병목**: 카피라이터와 디자이너 모두 최종 확인을 AM에게 의존 — AM이 단일 실패 지점(single point of failure).
3. **순차적 워크플로의 경직**: 카피 → 디자인 순서가 암묵적 규칙이 되어 있어 병렬 작업 불가. 카피 지연이 디자인 지연으로 연쇄.
4. **AI 툴 이미 도입, 단 컨텍스트 없음**: 팀 대부분이 이미 AI 카피 툴 사용 중이나 브랜드별 컨텍스트 주입이 수작업이라 일관성 낮음.
5. **멀티브랜드 컨텍스트 전환 비용**: 같은 날 다른 브랜드 2~3개를 오가며 작업 시 "톤 리셋" 시간 소요 및 혼선 리스크 — 브랜드 수가 늘수록 비례해서 악화.

## 5. 설계 권고사항 (리서치 기반)

1. **브랜드 룸 = 살아있는 컨텍스트 허브**: 정적 PDF·Notion 대체. 예시 카피, 톤 기준, 비주얼 에셋이 한 공간에 — AM이 업데이트하면 팀 전체에 즉시 반영.
2. **브랜드별 AI 컨텍스트 자동 주입**: 카피라이터가 브랜드 룸을 선택하면 해당 브랜드 가이드라인이 AI 프롬프트에 자동 탑재. 수작업 복붙 제거.
3. **카피-비주얼 동시 브리프 구조**: 카피 완성 후 디자이너에게 전달 시 텍스트 길이 제약·감정 톤·비주얼 레퍼런스가 함께 구조화되어 전달.
4. **Draft→Review→Approved 가시화**: AM 병목을 줄이기 위한 명시적 콘텐츠 상태 관리 — 누가 보고 있는지, 무엇이 블록인지 팀 전체가 볼 수 있어야 함.
5. **멀티브랜드 전환 UX**: 브랜드 간 이동 시 현재 활성 브랜드를 명확히 표시하는 컨텍스트 인디케이터 — 톤 혼선 방지.
